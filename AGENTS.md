## Quick Repository Facts

- **Framework & Language**: Flutter & Dart
- **Architecture**: Clean Architecture with feature-first presentation layer. Core contains domain entities, repositories, services, Riverpod providers, and themes. Features contain UI screens and widgets.
- **State Management**: Riverpod with code generation (`riverpod_generator`, `build_runner`)
- **External Services**: TMDB API for movie data, Firebase for authentication and real-time database
- **Important Generated Files**: `lib/firebase_options.dart` (FlutterFire CLI), Riverpod `.g.dart` files (build_runner)

## Commands and Setup

### 1. Clone and Install Dependencies
```bash
git clone <repository-url>
cd <project-folder>
flutter pub get
```

### 2. Environment Setup
Create a `.env` or `.env.local` file in the root directory with:
```text
TMDB_API_KEY=your_secure_api_key_here
```
- Contact the project lead for secure keys.
- Both `.env` and `.env.local` are git-ignored.

### 3. Firebase Configuration
Firebase configuration is generated via FlutterFire CLI. If Firebase settings change, run:
```bash
flutterfire configure
```
- Ensure FlutterFire CLI is installed (`dart pub global activate flutterfire_cli`).
- This regenerates `lib/firebase_options.dart`. Do not edit this file manually.

### 4. Generate Icons
App icons are generated via `flutter_launcher_icons`. After updating `assets/images/veto-icon.png`, run:
```bash
flutter pub run flutter_launcher_icons:main
```

### 5. Code Generation (Riverpod)
After adding or modifying Riverpod providers, generate code with:
```bash
flutter pub run build_runner build
```
- Use `--delete-conflicting-outputs` flag if needed.
- Generated files are `.g.dart` in the same directory.

### 6. Run the App
Ensure an emulator or physical device is connected:
```bash
flutter run
```

## Architecture and Development Notes

- **Clean Architecture Separation**:
  - `lib/core/domain/entities/` – Data models (Room, Movie, Vote, etc.)
  - `lib/core/domain/repositories/` – Abstract repository interfaces
  - `lib/core/domain/services/` – Business logic services
  - `lib/core/data/repositories/` – Concrete implementations (Firebase, TMDB)
  - `lib/core/providers/` – Riverpod providers (state management)
  - `lib/core/themes/` – UI themes and colors
- **Feature-First UI Layer**:
  - `lib/features/` contains UI screens and widgets grouped by feature (`voting`, `rooms`, `resolution`)
  - Each feature directory contains `screens/`, `widgets/`, and `services/` (feature-specific services)
- **Generated Files**: `lib/firebase_options.dart` and Riverpod `.g.dart` files are generated; do not edit manually.

## Commit Conventions

Follow the commit message structure:
```
<type>: <short summary>
```

**Types:**
- `feat:` New feature (e.g., `feat: add Google sign-in`)
- `fix:` Bugfix (e.g., `fix: null pointer in movie details`)
- `docs:` Documentation changes only (e.g., `docs: add TMDB setup guide`)
- `chore:` Maintenance or dependencies (e.g., `chore: upgrade Dart SDK`)
- `style:` Non-functional formatting changes.
- `refactor:` Code adjustments that don't add/alter behavior.
- `test:` Test additions or updates.

Examples:
- ✅ `feat: implement infinite scroll in home view`
- ✅ `fix: handle null release dates`
